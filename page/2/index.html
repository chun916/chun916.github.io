栈与队列：胡同

Problem

有一个死胡同，宽度刚好只能让一辆汽车通过，偏偏老有汽车开到死胡同来，这下麻烦了，最先开来的汽车要最后才能倒退出去。给定一个汽车开来的序列和一个可能的倒车出去的序列，请判断汽车能否都倒退出去，若能则输出Yes，否则输出No。

Input

首先输入一个整数T，表示测试数据的组数，然后是T组测试数据。每组测试数据首先输入一个正整数n（n≤10），代表开来的汽车数，然后输入2n个整数，其中，前n个整数表示汽车开来的序列，后n个整数表示汽车可能倒出的序列。

Output

对于每组测试，判断能否倒车出该死胡同，若能则输出“Yes”，否则输出“No”。引号不必输出。

  • inline(内联)函数：函数若在class body内定义完成，便成为inline候选人，优点：速度较快，但某些复杂函数无法在class body内完成,是否成为inline function 由编译器决定。
  • 上面的函数是常量成员函数，即，该函数不改变类中成员数据的内容，其中const的位置是在参数列表后，即在()后面，{}前面。在确定函数不改变数据内容的时候，一定要将函数写成常量成员函数的形式，即一定要加const。因为常量类只能调用常量成员函数，若函数不加const，则常量类调用该函数的时候就会出现错误，因为调用非常量成员函数，会改变类的成员数据，即与常量类自相矛盾。
  • access level（访问级别）
  1. public 供外界访问和使用，如外界使用的函数
  2. private 只在class内可用，比如数据部分
  3. protected 受保护的
  • 构造函数 constructor（ctor）
  1. 函数名一定等于类名
  2. double r = 0, double i = 0 ：默认实参，创建函数时没有传参时调用
  3. 没有返回值类型，因为构造函数用来创建对象
  4. : re (r), im (i) //初始值，初始列，构造函数的独有语法，和大括号内的赋值是不同阶段，构造函数中要尽量使用值的初始化，而不是赋值，这样性能会更高。
    大括号中同样可以赋值,但会放弃初始化的阶段，效率低了些。
    构造函数也可放在类的private区域，此时使用构造函数创建类将是错误的，因为private不能被访问到，即不允许外界通过构造函数创建类，单例模式即是如此。
  5. 不带指针的类多半不用写析构函数
  6. ctor 可以有很多个–overloading(重载)
  7. 防止函数重载调用冲突
  8. 构造函数放在private区，是一种单例模式（Singleton)。
  • const
    class类内的函数有两种：会/不会改变数据内容，不会改变数据内容的函数要在()后{}前加上const。
  • 参数的传递：pass by value vs pass by reference(to const) (按值传递或引用传递)
  1. 尽量不用按值传递，尤其是数据占用字节数较多时，值传递会将参数拷贝并传递，最好所有的参数都使用引用传递，引用底层就是指针。
  2. 引用传递形参会改变实参，通过const防止修改数值。

  • 返回值传递： return by value vs return by reference(to const)
    返回值的传递尽量使用引用传递

  • 函数内新创建的变量，函数结束后会销毁所占内存空间，不能使用引用传递。

  • friend（友元）
    通过友元类、友元函数访问类的私有成员变量。

    操作符重载和临时对象

  • c++中操作符就是函数，可以由设计者设计操作符功能

  • 操作符重载1-成员函数

  1. 任何成员函数又有一个隐藏的this pointer，指向调用者。
  2. 传递者无需知道接收者是以reference形式接收返回值， 即函数的使用者不必在意函数返回值的传递是 return by value or return by reference(to const)

栈：括号匹配

Problem

输入一个由中、小括符组成的字符串。判断其匹配情况，若括号可以成功匹配，则输出yes，否则输出no。

Input

测试数据有多组，处理到文件尾。对于每组测试，输入一个由中、小括符构成的字符串（长度不超过30）。

Output

对于每组测试，若括号可以成功匹配，则输出“yes”，否则输出“no”。引号不必输出。

  • 注：考虑（（（ 、 ））） 、([))等情况
  • 遍历字符串，左括号进栈，右括号和栈顶元素匹配，若匹配上，探出栈顶元素，否则不匹配；
  • 全为((( ，栈不为空，state==true;全为))) ，栈为空，state==false。

什么是队列？队列的特点是什么？

队列是一种操作受限的线性表，只允许在表的一端进行插入，而在表的另一端进行删除。
队列的特点是先进先出。

什么是循环队列？为什么要构成循环队列？

顺序存储结构的队列会出现 假溢出 的情况，通过循环队列可以解决顺序队列的缺点。
循环队列是将顺序队列臆造为一个环状的空间，即把存储队列元素的表从逻辑上视作一个环。
当首指针Q.front=MaxSize-1后，再前进一个位置就自动到0，用除法取余（%）运算实现。

如何判断循环队列的队空、队满？

front==rear 无法区分队满队空
队满：牺牲一个元素空间，当(rear+1)%MaxQSize==front时，队满
对空：front==rear

循环队列中的元素个数如何得到？

m:队列的容量
元素个数=(rear-front+m) % m

数据结构笔记（1）：基本概念和算法复杂度

Posted on 2021-01-26 Edited on 2021-05-06 In 数据结构笔记

数据结构基本概念

数据元素 数据项 数据对象

  • 数据元素是数据的基本单位，用于完整的描述一个对象，如一名学生记录。
  • 数据项是组成数据元素的、有独立含义的、不可分割的最小单位，如学生基本信息表的学号、姓名。
  • 数据对象是性质相同的数据元素的集合，是数据的一个子集。

数据结构

  • 什么是数据结构？
  • 数据结构是相互之间存在一种或多种特定关系的数据元素的集合，结构就是指数据元素之间的关系。
  • 数据结构学习的三个层次？
  1. 数据结构的基本知识和概念
  2. 数据结构的应用
  3. 算法的实现

  • 数据结构包括逻辑结构和存储结构，是问题的数据模型

    数据结构的三要素s

  • 数据的逻辑元素

  • 数据的存储结构

  • 数据的运算

    逻辑结构

    数据的逻辑结构从逻辑上描述数据，与数据的存储无关。

    存储结构

    存储结构又称物理结构，把数据对象存储到计算机时，既要求存储数据元素的数据，又要存储数据元素之间的逻辑关系。
    存储结构包括顺序存储结构和链式存储结构。

  • 顺序存储结构：借助元素在存储器的相对位置表示数据元素之间的逻辑关系，所有元素依次存放在一片连续的存储空间内。

  • 链式存储通过给每个节点附加指向后继元素地址的指针表示节点间的关系。

    算法和算法分析

    算法的定义

    算法是为了解决某类问题而规定的一个有限长的操作序列。

    算法的特性

  • 有穷性：执行有穷步且在又穷时间内完成。

  • 确定性：每种情况所执行的操作，在算法中都有确定的规定，不会产生二义性。

  • 可行性

  • 输入： 有零个或多个输入

  • 输出： 有一个或多个输出

    算法的时间复杂度

  • 影响算法时间代价的最主要因素是问题规模，问题规模是算法求解问题输入量的多少，是问题大小的本质表示。

  • 一条语句的重复执行次数称作语句频度，一个算法的执行时间可用该算法中所有语句频度之和来度量。

  • 常量阶时间复杂度：与问题的规模无关，执行时间恒定的算法。
    for(i=0;i<1000;i++){x++;s=0;}
    算法的执行时间为常数，即使常数再大，时间复杂度都是O(1)。

    算法的空间复杂度

    空间复杂的度量算法的存储空间，也是问题规模n的函数，一般只需分析算法在实现时所需的辅助空间。

  • 算法原地工作：算法执行所需辅助空间相对输入量而言是常数，空间复杂度为O（1）。

    只有一个辅助变量t，与问题规模n无关，空间复杂度为O（1）。
